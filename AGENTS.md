# AGENTS.md

> **核心目标**：交付高质量、安全、可维护的代码。  
> **强约束**：所有用户可见输出、文档、注释与提交信息统一使用简体中文。  
> **默认取向**：性能优先，先分析后执行，保持目录清晰、代码整洁。  

## 1. 工作原则

- 先分析后执行：先明确目标、约束、边界、风险与验收标准，再开始改动。
- 方案要能落地：优先选择简单、可维护、能解决根因的方案，避免堆补丁和过度设计。
- 安全优先：禁止硬编码密钥；所有外部输入都必须做校验、清洗和边界检查。

## 2. 编码要求

- 文件统一使用 UTF-8 编码，不使用 BOM。
- 标识符使用清晰英文命名，避免无意义缩写。
- 关键逻辑、接口约束和不明显的业务判断需要补充简体中文注释，重点解释“为什么”。
- 设计遵循 `KISS`、`YAGNI`、`SOLID`、`DRY`。

## 3. 变更与验证

- 逻辑变更必须补充自动化测试。
- 每次修改都要执行受影响范围内的最小充分验证，例如单测、集成测试、契约测试、端到端测试或静态检查。
- 纯文档或注释更新可以不跑测试，但交付时必须说明原因。
- 交付时需要说明：做了什么、为什么这样做、如何验证、剩余风险和后续建议。

## 4. 检索与工具约束

- 本地检索优先使用 `rg`。
- 除纯本地微调外，默认优先查官方文档、标准规范和权威资料。
- 对于联网得到的关键结论，尽量保留来源依据。

## 5. 项目级偏好

- 大模型负责联网搜索、归纳和整理；TypeScript 负责执行工具、校验、修复和排程。
- 优先保证主链路稳定，再处理风格层优化。
- 对历史遗留逻辑，优先删除死代码和错误文案，再考虑扩展新能力。
