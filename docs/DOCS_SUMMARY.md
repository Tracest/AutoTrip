# AutoTrip 文档总览

这份文件是仓库内项目文档的索引页。

范围说明：

- 包含：项目根目录与 `docs/` 目录下的项目自有 Markdown 文档
- 不包含：`node_modules/` 中的第三方文档

## 推荐阅读顺序

1. `README.md`
2. `docs/RUNBOOK.md`
3. `docs/PROJECT_STATUS.md`
4. `docs/ARCHITECTURE.md`
5. `docs/TROUBLESHOOTING.md`

## 文档清单

### `README.md`

用途：

- 说明项目是什么
- 说明当前默认行为
- 说明如何启动、登录、配置模型和实际使用
- 给出环境变量、命令和最小可用路径

适合什么时候读：

- 第一次接手项目
- 想把项目跑起来
- 想知道“现在到底应该怎么用”

### `docs/PROJECT_STATUS.md`

用途：

- 记录当前版本已经完成的关键工作
- 说明当前默认数据路径和验证结果
- 汇总已知边界和后续建议

适合什么时候读：

- 想快速了解项目做到哪一步
- 想做版本收尾或 handoff
- 想知道“哪些问题已经解决，哪些还没解决”

### `docs/RUNBOOK.md`

用途：

- 说明项目的标准启动顺序
- 说明首次登录、模型配置和最小烟雾测试
- 提供日常运行和健康检查清单

适合什么时候读：

- 你要把项目跑起来
- 你要做交付自检
- 你需要一份比 README 更偏运维动作的手册

### `docs/ARCHITECTURE.md`

用途：

- 解释系统结构
- 解释认证、模型配置、候选点来源和规划流水线
- 解释工作区保存与重排机制

适合什么时候读：

- 要改规划逻辑
- 要接 API 或重构模块
- 要排查“为什么会走这条数据路径”

### `docs/TROUBLESHOOTING.md`

用途：

- 汇总运行和配置问题的快速修复方法
- 覆盖 Windows PowerShell、PostgreSQL、Ollama、模型配置和规划结果问题

适合什么时候读：

- 跑不起来
- 登不上
- 模型测不通
- 规划结果异常

## 当前文档重点结论

- AutoTrip 当前是本地优先的单用户工作台
- 默认推荐本地 Ollama，不强制要求远程模型服务
- `AMAP_API_KEY` 现在是增强项，不是最低可用前提
- 不配置地图 Key 时，默认候选点来源是 Wikimedia 在线公开资料
- 系统不再用 mock / 示例 POI 假装规划成功
- 规则排程已经具备更强的时段意识，中段更偏美食，收尾更偏夜景
- 如果已经有可执行的在线候选点，本地 Ollama 结构化细化失败不会再默认制造很吵的失败体验
- 文档已经覆盖了“是什么、怎么跑、现在做到哪、出问题怎么修”这四类最核心问题

## 如果你只想做一件事

- 想运行项目：先看 `README.md`
- 想按步骤启动和验收：先看 `docs/RUNBOOK.md`
- 想知道项目现状：先看 `docs/PROJECT_STATUS.md`
- 想改代码：先看 `docs/ARCHITECTURE.md`
- 想解决故障：先看 `docs/TROUBLESHOOTING.md`
