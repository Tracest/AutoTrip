# AutoTrip 项目状态

更新时间：2026-04-08

这份文件用于项目收尾、交接和后续迭代时快速确认“当前到底完成到了哪里”。

## 当前版本结论

项目已经从“依赖地图 Key 或模型兜底示例数据”的状态，切换到了“本地优先、联网候选点优先、失败时诚实报错”的状态。

一句话概括：

- 现在它已经能在不配置 `AMAP_API_KEY` 的情况下，结合 Wikimedia 在线资料和本地 Ollama，生成一份可继续编辑的中文行程草案。

## 已完成的关键改动

### 1. 模型接入体验

- 默认推荐本地 Ollama
- 本地 Ollama 可以不填真实 API key
- 已支持模型发现接口
- 已支持连接测试
- 已统一保存 OpenAI-compatible 配置

### 2. 候选点来源策略

- 有 `AMAP_API_KEY` 时优先走高德地图
- 没有高德 Key 时默认走 Wikimedia 在线公开资料
- `AUTO_TRIP_FORCE_MOCK=1` 仅保留给测试 / 显式 mock 场景
- 不再把 mock 示例 POI 当成正常规划结果保存

### 3. 候选点质量控制

- 对片区、道路、泛概念、学校、普通楼宇等条目做了更强过滤
- 增加了更像真实景点的信号识别
- 对经典城市景点和老字号做了额外加权
- 改善了国内城市的中文输出优先级

### 4. 规则排程质量

- 不再机械切片
- 更偏“上午主景点 / 中段美食 / 收尾夜景”
- 会尽量避免同类项目连续堆叠
- 即使模型细化失败，也能保留一份可执行的规则行程

### 5. 用户体验与提示

- 没有可信候选点时明确报错
- 不再默默保存示例数据
- 当已有可执行在线候选点时，本地 Ollama 的结构化细化失败不再默认制造强噪音 warning

## 当前默认推荐使用方式

推荐的最小可用路径：

1. 启动 PostgreSQL
2. 初始化 Prisma
3. 启动 Ollama
4. 拉一个本地模型
5. 启动 Next.js
6. 登录创建管理员
7. 配置 `http://127.0.0.1:11434/v1`
8. 不配 `AMAP_API_KEY` 直接规划

推荐模型：

- `qwen2.5:3b`
  跑通流程更轻更快
- `deepseek-r1:8b`
  通常比更小模型更稳

## 当前已知边界

### 数据源边界

- Wikimedia 是公开资料源，不是地图级事实源
- 开放时间、票务、精确通勤仍可能不完整
- 城市级热度排序仍偏启发式

### 模型边界

- 本地小模型的结构化 JSON 稳定性有限
- 即使接口通了，也可能返回空或不完整结果
- 因此系统仍必须依赖规则草案作为稳定底座

### 国际目的地边界

- 当前国际目的地仍属于 beta 路径
- 整体策略更偏国内中文目的地
