## 最近完成的关键变化

### 1. 主搜索职责切换

之前：

- TypeScript 默认负责网上搜索与初筛
- 模型更多是后置润色

现在：

- 模型自己决定搜索词与抓取页面
- TypeScript 负责执行工具、接收结构化结果并做校验修复

### 2. 候选点稳定性补强

最近补齐了以下问题：

- “贵州省”“贵阳轨道交通 1 号线”“贵阳广播电视台”这类伪 POI 过滤
- `花果园双子塔` 这类真实地标的异常坐标修复
- 模型联网后兴趣覆盖不足时，自动并入城市种子点
- `贵州省博物馆` 被误标成美食之类的类别纠偏
- `甲秀楼` / `甲秀楼夜景` 这类同源变体去重

## 当前主要限制

- 目的地仍受支持城市列表限制
- 无高德时通勤和营业时间仍是弱地图级精度
- 模型联网质量依赖公开网页质量与模型能力
- 小模型更容易出现候选点覆盖不足或结构化输出不稳

## 下一批值得做的事

- 给候选点增加来源可信度分层
- 优先使用官网、百科、权威目录站做加权
- 加强营业时间与营业状态验证
- 做更细的区域聚类，减少单日跨城式排程
